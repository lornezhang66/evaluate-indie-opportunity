# 快速评估示例

这些示例展示 Skill 的决策方式，不代表固定市场结论。真实使用时，应根据当前证据重新评估。

## 示例一：热门开源项目套壳 SaaS

### 输入

> 我发现一个近期很火的 AI 开源项目，Star 增长很快。我想做一个中文 UI，加上登录和订阅，值得做吗？

### 典型判断

- **类别**：`CASH EXPERIMENT` 或 `STRATEGIC OPTION`
- **动作**：`RESEARCH`
- **证据等级**：E0–E1
- **核心理由**：Star 证明开发者关注，不证明目标用户愿意为中文 UI 和托管付费。

### 第一笔交易

先向一个明确人群出售固定范围的安装、配置、私有部署或结果代办服务；不要先建设通用 SaaS。

### Phase 0

- 假设：某类用户愿意为“无需理解底层项目即可获得目标结果”付费。
- 行动：制作一次结果演示，向 10 位符合画像的人提出真实报价。
- 成功：出现付款、押金或明确资源承诺。
- 停止：只有点赞、收藏和“上线通知我”，没有承诺。

## 示例二：已有自然下载的开源插件

### 输入

> 我的插件已经进入应用市场，并获得了一些自然下载。我要不要马上开发会员体系？

### 典型判断

- **类别**：`COMPOUNDING ASSET`
- **动作**：`TEST NOW`
- **证据等级**：E2
- **核心理由**：自然下载是采用证据，但尚未证明留存和付款；会员体系会提前增加复杂度。

### 第一笔交易

从安装、配置、专业模板、跨工具接入或明确高级结果中选一个最小报价，直接面向高频使用者验证。

### Phase 0

- 假设：至少一类用户不仅下载，而且反复使用，并愿意为一个高价值结果付费。
- 行动：增加轻量反馈入口，访谈活跃用户，并提出一个无需会员系统即可成交的报价。
- 成功：出现真实付款或重复使用证据。
- 停止：下载增长但无人持续使用，也没有主动反馈。

## 示例三：用小游戏赚第一笔互联网收入

### 输入

> AI 已经能快速生成小游戏，我想用两个周末做一个游戏，尝试赚到第一笔广告收入。

### 典型判断

- **类别**：`CASH EXPERIMENT`
- **动作**：`TEST NOW`
- **证据等级**：E0
- **核心理由**：市场证据弱，但实验时间和损失可以封顶，因此可以作为学习实验，不能自动升级成正式主线。

### 第一笔交易

第一阶段不要把“提交平台”当成交易。先确认平台主体、备案、广告和审核路径，再验证玩家是否主动重复游玩。

### Phase 0

- 假设：一个核心循环和一个差异点足以让目标玩家主动重玩。
- 行动：只做可玩原型，邀请真实玩家测试，不建设账号、剧情、排行榜和商城。
- 成功：达到实验前设定的完成、主动重玩或次日返回阈值。
- 停止：两周结束仍没有重复行为，或者合规与分发成本超出上限。

## 比较多个机会时

不要只比较最终分数。先区分它们承担的任务：

- 现金实验：能否快速完成真实交易？
- 复利资产：能否积累留存、用户、代码、内容、数据或渠道？
- 战略期权：是否符合长期结构化优势，并值得低成本保留？

最终只选择一个当前行动对象和一个观察对象，其余进入停车场。
